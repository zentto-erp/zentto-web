## GAPS CRÍTICOS — Lo que falta para competir en grande

### PRIORIDAD 1: Seriales, Lotes y Albaranes (Logística de salida)

**Problema:** Cuando un comercio vende productos con serial (electrónica, electrodomésticos, equipos médicos), necesita:
1. Registrar serial al ingresar mercancía
2. Seleccionar serial al vender
3. Generar **albarán/nota de entrega** con los seriales
4. Rastrear garantía por serial

**Lo que necesitamos:**

| Componente | Descripción |
|-----------|-------------|
| Tabla `inv.ProductSerial` | Serial individual vinculado a artículo + almacén |
| Tabla `inv.ProductLot` | Lote con fecha producción/vencimiento |
| Documento ALBARAN / NOTA_ENT | Documento de salida de mercancía (ya existe tipo NOTA_ENT pero sin lógica de seriales) |
| Hoja de salida de almacén | PDF con seriales entregados, firmado por quien retira |
| Trazabilidad | Desde serial → compra origen → venta destino → cliente |

### PRIORIDAD 2: Recepción de mercancía y devoluciones

**Problema:** Compras necesita un flujo completo:
```
Orden de Compra → Recepción parcial/total → Verificación calidad → Ingreso almacén → Factura proveedor
```

Actualmente: OC → Compra directa. No hay paso intermedio de recepción ni verificación.

**Lo que necesitamos:**
- Documento de recepción de mercancía (goods receipt)
- Recepción parcial (recibir 50 de 100 unidades)
- Devolución a proveedor (goods return)
- Inspección de calidad en recepción

### PRIORIDAD 3: Métodos de valoración de inventario

**Problema:** Sin FIFO/LIFO/Promedio no se puede calcular correctamente el costo de ventas.

**Lo que necesitamos:**
- Configuración por artículo: FIFO, LIFO, Promedio ponderado, Último costo
- Cálculo automático del costo de venta al facturar
- Reporte de valoración de inventario por método

### PRIORIDAD 4: Proveedores de transporte y guías de despacho

**Problema:** Empresas que despachan mercancía necesitan:
1. Registro de transportistas (empresa, conductor, vehículo, placa)
2. Guía de despacho vinculada a factura
3. Confirmación de entrega (POD)
4. Seguimiento de envíos

**Lo que necesitamos:**
- Maestro de transportistas
- Documento GUIA_DESPACHO vinculado a FACTURA + NOTA_ENT
- Estado de entrega: Despachado → En tránsito → Entregado
- Firma digital de recepción

### PRIORIDAD 5: Control de flota (sub-módulo de activos fijos)

**Problema:** Empresas con vehículos propios necesitan:
1. Registro de vehículos (ya existe `sp_crud_vehiculos.sql` como maestro)
2. Control de mantenimiento preventivo/correctivo
3. Control de combustible
4. Kilometraje y rutas
5. Documentos legales (seguro, revisión técnica, permisos)
6. Asignación de conductores

**Ya existe:** Tabla vehiculos como maestro + campos VehiclePlate/Mileage en documentos de venta.
**Falta:** Todo el sub-sistema de gestión de flota.

### PRIORIDAD 6: CRM básico

**Problema:** Sin CRM el departamento comercial no tiene herramientas.

**Lo que necesitamos:**
- Pipeline de ventas (Lead → Oportunidad → Cotización → Venta)
- Actividades/seguimiento por cliente
- Pronóstico de ventas
- Histórico de interacciones

**Ya existe:** Tabla `sys.Lead` + `usp_sys_Lead_Upsert`. Falta el frontend y pipeline.

### PRIORIDAD 7: Manufactura básica (BOM)

**Para empresas que fabrican:**
- Lista de materiales (BOM)
- Órdenes de producción
- Consumo de materia prima → Producto terminado
- Costeo de producción

### PRIORIDAD 8: Permisos granulares (RBAC)

**Problema:** Actualmente es admin/no-admin. Un ERP profesional necesita:
- Roles personalizables
- Permisos por módulo + acción (ver/crear/editar/anular)
- Permisos por sucursal
- Restricciones de precio (descuento máximo, precio mínimo)
